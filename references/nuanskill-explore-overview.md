## 工作流

### 实时查询模式（用户主动询问）

```
1. Chrome（`{USER_DATA_DIR}chrome-profile\`）访问 https://myl.nuanpaper.com/tools/journal（无头 + Cookie 注入）
2. 切换到「探索总览」标签页（第 2 个标签）
3. 提取各区域的流转之柱、奇想星、灵感露珠、子区域收集项数据
4. 按区域分组汇报 X/Y 格式的进度
5. 将数据保存到 {USER_DATA_DIR}nuan_explore.json（覆盖更新）
```

### 定时更新模式（定时任务触发）

```
1. Chrome（`{USER_DATA_DIR}chrome-profile\`）访问 https://myl.nuanpaper.com/tools/journal（无头 + Cookie 注入）
2. 切换到「探索总览」标签页
3. 提取所有区域的探索数据
4. 保存到 {USER_DATA_DIR}nuan_explore.json
5. 不主动汇报，仅更新本地缓存
```

> **关键**：实时查询时，Agent 应在回答用户问题的同时，将当前网页的所有探索数据保存到本地，即使部分数据不在用户询问范围内。
>
> **常识参考**：各区域固定信息（收集项用途、幻境类型、区域划分）见 `references/nuanskill-game-knowledge.md` →「大世界探索内容」。
>
> **写入格式**：必须严格按照 `SKILL.md` →「数据文件格式规范」→ `nuan_explore_data.json` 的字段结构写入，`updated_at` 为 ISO 时间格式。

## 页面数据提取

### DOM 结构参考

> **注意**：以下为参考结构，Agent 应根据实际页面动态调整选择器。

```
# 区域卡片
.region-card
  .region-name ← 区域名称（如「心愿原野」）
  .stat-item
    .stat-label ← 维度名称（如「流转之柱」）
    .stat-progress ← X/Y

# 子区域收集项
.sub-region
  .sub-region-name ← 子区域名称
  .collection-item
    .collection-name ← 收集项名称
    .collection-progress ← X/Y
```

### 数据提取伪代码

```powershell
# 切换到探索总览标签页
# 等待页面加载完成

# 提取各区域数据
$regions = @{}
$regionCards = $page | QuerySelectorAll ".region-card"

foreach ($card in $regionCards) {
    $regionName = $card | QuerySelector ".region-name" | GetText
    $regionData = @{
        "流转之柱" = ExtractProgress($card, "流转之柱")
        "奇想星" = ExtractProgress($card, "奇想星")
        "灵感露珠" = ExtractProgress($card, "灵感露珠")
        "子区域" = ExtractSubRegions($card)
    }
    $regions[$regionName] = $regionData
}

# 保存 to {USER_DATA_DIR}nuan_explore.json
Save-Json "{USER_DATA_DIR}nuan_explore.json" @{
    "updated_at" = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"
    "regions" = $regions
}
```

## 错误处理

| 症状 | 诊断方向 | 参考 |
|------|---------|------|
| 页面加载失败 | Cookie 过期 → 走「首次初始化」流程 | `SKILL.md` → 首次初始化 |
| 数据提取失败 | DOM 结构变化 → 动态调整选择器 | 本报告 → 页面数据提取 |
| 区域名称不匹配 | 游戏版本更新 → 动态提取，不硬编码 | 本报告 → 数据维度 |
| 子区域数据缺失 | 子区域未展开 → 模拟点击展开 | 本报告 → 工作流 |

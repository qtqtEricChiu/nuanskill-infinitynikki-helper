# 奇想手账日程监控 — Agent 执行参考

> **数据源**：https://myl.nuanpaper.com/tools/journal（日程便利贴标签页）
> **依赖**：已登录的浏览器会话（cookie 见 user-data/nuan_user_profile.json）

## 前置条件

Browser agent 必须使用已保存 cookie 的浏览器会话访问页面，确保携带有效认证信息（`momoToken`、`momoOpenid` 等）。

## 1. 体力监控

### 系统参数

| 参数 | 数值 |
|------|------|
| 体力上限 | 350 点 |
| 恢复速率 | 每 5 分钟恢复 1 点 |
| 满体力恢复时间 | 缺口点数 × 5 分钟 |

### 页面数据提取

在日程便利贴标签页中提取：

- **活跃能量**：格式如 `197/350`，提取当前值 X
- **剩余恢复时间**：格式如 `HH:MM`，距离满体力的倒计时

### 触发场景

#### 场景 A — 清体力检测

**触发**：本次体力值相比 `last_stamina` 大幅减少（用户已清体力）。

**语义区分（关键）**：

| 体力状态 | 含义 | 动作 |
|---------|------|------|
| X 接近 0（如 3/350） | 刚清完体力，缺口大，有充足恢复时间 | 计算满体时间，预告下次清体力窗口 |
| X 接近 350（≥344） | 体力即将回满，需要尽快清 | 走场景 B 催促清体力 |

> **注意**：体力 3/350 不是"快满了"，而是"刚清完"。缺口 347 点 ≈ 28 小时 55 分钟才能回满。

**动作**：
1. 计算新满体力预计时间：`恢复时间 = (350 - X) × 5 分钟`
2. 通过微信提醒：`请在 [满体力预计时间] 前清体力`
3. 重置 `predicted_full_time`，取消旧提醒

#### 场景 B — 即将满体力提醒

**触发**：
- 距离满体力 ≤ 30 分钟（体力 ≥ 344）
- 体力已达 350（满体力）

**动作**：微信发送"体力即将满，请及时清体力"或"体力已满，请及时清体力"。

### 状态变量

| 变量 | 说明 |
|------|------|
| `last_stamina` | 上次记录的体力值（整数） |
| `predicted_full_time` | 预计满体力的时间点 |

### 汇报格式

```
【奇想手账体力提醒】当前体力 X/350，缺口 Y 点，预计 [日期时间] 恢复满格。请在 [时间] 前清体力～
```

## 2. 美鸭梨挖掘

### 数据来源

同一页面的"美鸭梨挖掘"模块（需点击展开）。

### 项目清单

**从页面动态提取**（项目名称和数量由游戏内调整，不可硬编码）。

| 字段 | 说明 |
|------|------|
| 项目名称 | 页面动态显示，从 DOM 直接提取 |
| 状态 | `努力挖掘中！` / `已完成` |
| 剩余时间 | 倒计时（如 XX小时XX分钟） |
| 总进度 | `已完成数 / 总数` |

### 汇报格式

```
【美鸭梨挖掘进度】总进度 X/Y

| 项目 | 状态 | 剩余时间 |
|------|------|----------|
| {项目1名称} | 努力挖掘中！ | XX小时XX分钟 |
| {项目2名称} | 已完成 | — |
```

**合并规则**：多个未完成项目剩余时间一致时可合并，如"剩余 3 个项目均预计 X 小时后完成"。

### 时间计算

根据剩余倒计时 + 当前系统时间（`Get-Date`）推算各项目预计完成时间。

## 3. 心之突破幻境（周本）

### 数据来源

同一页面"日程便利贴"中的"心之突破幻境"模块。

### 系统参数

| 参数 | 说明 |
|------|------|
| 刷新周期 | 每周刷新一次 |
| 状态字段 | `已挑战` / `未挑战` |
| 当前 Boss | 动态（如 幻象愿望大师·奇格格达） |
| 刷新倒计时 | 距离下次刷新的剩余时间（如 2天6小时） |

### 汇报格式

```
【心之突破幻境 · 周本状态】

| 项目 | 内容 |
|------|------|
| 状态 | 已挑战 |
| 当前 Boss | 幻象愿望大师·奇格格达 |
| 下次刷新 | 约 X 天 X 小时后（日期） |
```

### 触发场景

| 场景 | 触发条件 | 动作 |
|------|---------|------|
| 每周提醒 | 接近刷新时间且状态为未挑战 | 微信提醒「周本尚未挑战，请尽快完成」 |
| 状态查询 | 用户主动询问 | 按汇报格式输出当前状态 |

## 4. 朝夕心愿（每日心愿 / 日活）

### 数据来源

同一页面"日程便利贴"中的"朝夕心愿"模块。

### 系统参数

| 参数 | 说明 |
|------|------|
| 刷新周期 | 每日刷新 |
| 进度格式 | `X/500`（当前积分/上限） |
| 完成判定 | 达到 500 即为完成 |

### 汇报格式

```
【朝夕心愿 · 日活状态】

| 项目 | 内容 |
|------|------|
| 状态 | 已完成 |
| 进度 | 500/500 |
```

### 触发场景

| 场景 | 触发条件 | 动作 |
|------|---------|------|
| 每日检查 | 每天首次查询 | 汇报当前进度，未完成时提醒 |
| 状态查询 | 用户主动询问 | 按汇报格式输出 |

---

## 5. 时间计算规则

> **强制**：涉及时间推算必须先执行 `Get-Date -Format "yyyy-MM-dd HH:mm:ss"` 获取系统准确时间，严禁依赖环境信息中的日期。

适用场景：体力满格推算、美鸭梨完成时间、任何"预计 X 分钟后"的计算。

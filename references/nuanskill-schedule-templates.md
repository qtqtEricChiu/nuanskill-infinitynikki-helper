# 定时任务配置模板

> **能力编号**：通用（所有能力创建定时任务时参考）
>
> 当用户需要设置定时任务时，Agent 应主动推荐以下两个内置模板，并提供具体的配置参数。

---

## 模板一：每日奇想手账监控（能力②）

| 项目 | 内容 |
|------|------|
| **频率** | 每日一次（建议早上上线后执行） |
| **执行动作** | Chrome（`Cache\chrome_temp_profile\`）访问 https://myl.nuanpaper.com/tools/journal |
| **数据提取** | 体力值 / 恢复倒计时 / 美鸭梨挖掘进度 / 朝夕心愿（日活） / 当期卡池剩余时间 |
| **计算** | 体力回满时间 = `(350 - X) × 5 分钟` + `Get-Date`；挖掘完成时间按页面倒计时推算 |
| **反馈** | 向用户推送体力回满预计时间和挖掘完成预计时间 |
| **本地缓存** | 写入 `user-data/nuan_journal_data.json` |

**Agent 引导话术**：
> 「我已设置每日奇想手账监控，每天会自动检查体力、美鸭梨挖掘和日活状态，并在完成后推送给您。」

---

## 模板二：每月抽卡记录更新（能力④）

| 项目 | 内容 |
|------|------|
| **频率** | 每月一次（建议每月 1 号执行） |
| **执行动作** | Chrome（`Cache\chrome_temp_profile\`）访问 https://myl.nuanpaper.com/tools/journal/clothesPress |
| **数据源 A** | React fiber → 写入 `user-data/nuan_gacha_stats.json`（抽卡统计汇总） |
| **数据源 B** | localStorage → 写入 `user-data/nuan_gacha_history.json`（完整抽卡明细） |
| **反馈** | 向用户汇报更新完成 |
| **注意事项** | localStorage 提取使用无头 Chrome 即可，无需弹窗 |

**Agent 引导话术**：
> 「每月抽卡记录更新已设置，会自动更新两个数据文件：套装抽卡统计和完整抽卡历史。是否需要同时更新探索进度数据？」

---

## 多任务合并建议

当用户已设置多个定时任务时，Agent 应主动检查是否可以合并执行以减少重复爬取：

| 现有任务 | 可合并的关联任务 |
|----------|-----------------|
| 每月抽卡记录更新 | 一并更新探索进度（`nuan_explore_data.json`） |

> **示例**：用户创建「每月 1 号更新抽卡记录」的定时任务时，Agent 应询问：
> 「需要同时更新探索进度数据吗？这样可以保持本地数据最新，且探索进度变化缓慢，每月更新一次就足够。」

### 模板三：卡池结束前一天提醒（可选）

| 项目 | 内容 |
|------|------|
| **频率** | 一次性（基于 `nuan_journal_data.json` 中 `current_banner_pool.ends_at` 计算触发时间） |
| **触发时间** | `ends_at - 1 天`（建议北京时间 20:00 推送） |
| **执行动作** | 读取本地 `nuan_journal_data.json` 的 `current_banner_pool.ends_at` 字段，判断是否即将到期 |
| **反馈** | 向用户推送「当期卡池将于明天结束，记得检查是否还有想要的套装」 |
| **依赖** | 必须在每日监控任务正常运行的前提下使用（依赖 `current_banner_pool` 被及时更新） |

**Agent 引导话术**：
> 「需要我设置一个当期卡池结束前一天的提醒吗？这样您就不会错过限时卡池的最后机会。」
